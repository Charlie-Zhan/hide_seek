# SKILL: 核心玩法系统

## 使用场景

当任务涉及阶段流转、身份轮换、伪装、拍击、遮挡、得分或局内规则时，阅读本文件。

---

## 1. 核心循环

```text
创建房间 -> 玩家就绪 -> 开始整场 -> 每名玩家轮流当找人者 -> 结算总分 -> 返回房间
```

单轮：

```text
Preview -> Hide -> Seek -> Result
```

---

## 2. 身份轮换

假设玩家顺序为：

```text
[A, B, C, D]
```

则找人者顺序为：

```text
Round 1: A
Round 2: B
Round 3: C
Round 4: D
```

玩家中途离开时：

- 如果未开始，重新排序。
- 如果已开始，尽量保持既定顺序；断线玩家可跳过。

---

## 3. 阶段规则

### Preview

- 地图显示原始状态。
- 玩家角色隐藏。
- 禁止输入。
- 目标：记忆地图。

### Hide

- 找人者不可见地图，只见黑屏或静态屏保。
- 找人者输入被忽略。
- 躲藏者正常速度移动。
- 躲藏者停止后自动伪装。
- 躲藏者可随时切换道具。

### Seek

- 找人者可移动和拍击。
- 躲藏者始终为道具形态。
- 躲藏者慢速移动。
- 躲藏者可随时切换道具。
- 找人者拍击次数有限。

### Result

- 展示本轮抓获数量、存活玩家、得分变化。
- 短暂停留后进入下一轮或整场结算。

## 4. 伪装系统

每张地图定义一个道具池：

```ts
interface DisguisePropConfig {
  id: string;
  displayName: string;
  spritePath: string;
  radius: number;
  category: 'box' | 'plant' | 'bucket' | 'chair' | 'food' | 'misc';
}
```

躲藏者拥有：

```ts
currentPropId: string
```

右键切换：

```text
currentPropId = next prop in map prop pool
```

无冷却，无资源消耗。

---

## 5. 自动伪装

Hide 阶段：

```text
正在移动 -> 显示角色
停止超过 0.2s~0.3s -> 显示当前道具
```

Seek 阶段：

```text
始终显示当前道具
移动时不变回角色
```

---

## 6. 遮挡物

躲藏者不是进入容器，而是通过地图视觉遮挡隐藏。

可用遮挡物：

- 草丛；
- 树冠；
- 柱子；
- 墙角；
- 大桌子；
- 大石头；
- 招牌。

遮挡物不提供交互按钮，只通过层级遮挡产生效果。

---

## 7. 可破坏道具与大障碍物

### 可破坏道具

- 木箱；
- 桶；
- 盆栽；
- 椅子；
- 小罐子；
- 食材篮。

它们可被找人者拍碎，也可作为躲藏者伪装目标。

### 不可破坏大障碍物

- 墙；
- 柱子；
- 灶台；
- 柜台；
- 大桌子；
- 树干。

它们主要承担地图结构、碰撞和遮挡。

---

## 8. 拍击系统

基础参数：

```text
angleDeg = 90
radius = 100~140 px
maxHits = hiderCount * 2
```

判定目标：

- 扇形内所有可破坏普通道具；
- 扇形内所有未被抓获躲藏者。

拍击结果：

- 普通道具：破碎并从地图中移除或变成 destroyed 状态。
- 躲藏者：被抓获，显示角色弹出或被揭穿效果。
- 拍空：只消耗次数。

---

## 9. 得分系统

```text
找人者抓到 1 人：+1
找人者抓到全部躲藏者：额外 +1
躲藏者存活到本轮结束：+1
```

一轮结束条件：

- 搜索时间归零；
- 找人者拍击次数归零；
- 所有躲藏者被抓。

---

## 10. 禁止事项

- 不要让伪装物自动抖动暴露。
- 不要加入扫描、冲刺。
- 不要加入容器躲藏。
- 不要给切换道具加冷却。
- 不要在 MVP 中加入小任务和随机事件。
