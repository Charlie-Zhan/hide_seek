# SPEC PHASE 01: 本地核心玩法原型

## 阶段目标

在单机环境中验证核心玩法：阶段流转、躲藏者伪装、找人者扇形拍击、有限拍击次数、基础计分。此阶段允许使用本地假玩家或调试控制，不需要联机。

## 功能范围

### 1. 本地玩家模型

支持 2~4 名本地测试玩家的数据模型：

```ts
LocalPlayer {
  playerId: string;
  displayName: string;
  role: PlayerRole;
  state: PlayerState;
  score: number;
  position: Vec2;
  facing: Vec2;
  currentPropId: string;
  captured: boolean;
}
```

MVP 调试方式：

- 1 名玩家由真实输入控制；
- 其他玩家可用静态假玩家、简单随机位置，或开发者热键切换控制对象。

### 2. 阶段流转

实现：

```text
Preview -> Hide -> Seek -> Result -> Next Round
```

阶段时间读取 `game_config.json`。

### 3. Preview 阶段

- 地图显示。
- 玩家角色隐藏。
- 输入不生效。
- HUD 显示“观察地图”。

### 4. Hide 阶段

- 如果当前控制者是找人者，显示黑屏或静态屏保。
- 找人者输入不生效。
- 躲藏者可正常速度移动。
- 躲藏者停止超过阈值后自动显示为当前道具。
- 躲藏者可右键无冷却切换道具。

### 5. Seek 阶段

- 找人者可移动。
- 找人者右键拍击前方扇形。
- 躲藏者始终以道具形态显示。
- 躲藏者慢速移动。
- 躲藏者可右键无冷却切换道具。

### 6. 扇形拍击

本地实现判定：

- 命中多个可破坏普通道具；
- 命中躲藏者；
- 扣除拍击次数；
- 拍击次数归零本轮结束；
- 全部躲藏者被抓本轮结束。

推荐初始参数：

```text
angleDeg = 90
radiusPx = 120
maxAttackCount = hiderCount * 2
```

### 7. 本地计分

- 找人者抓到每人 +1。
- 全抓额外 +1。
- 存活躲藏者 +1。
- Result 阶段展示本轮得分变化。

---

## UI 要求

Game HUD 至少显示：

- 当前阶段；
- 倒计时；
- 当前身份；
- 剩余拍击次数，仅找人者；
- 当前伪装道具，仅躲藏者；
- 本轮抓获数量；
- 总分。

---

## 调试工具

允许加入仅开发可见的调试项：

- 快进当前阶段；
- 切换控制玩家；
- 显示扇形拍击范围；
- 显示隐藏玩家位置。

但必须在代码中标记为 debug，后续微信发布不可默认显示。

---

## 验收标准

- [ ] Preview 阶段角色不可见且不能操作。
- [ ] Hide 阶段找人者黑屏且不能操作。
- [ ] Hide 阶段躲藏者移动正常，停下自动变道具。
- [ ] 躲藏者切换道具无冷却。
- [ ] Seek 阶段找人者可以移动并扇形拍击。
- [ ] 一次拍击能打碎范围内多个普通道具。
- [ ] 一次拍击能抓住范围内躲藏者。
- [ ] 拍击次数用完后本轮结束。
- [ ] 得分规则正确。
- [ ] 每个玩家能轮流当找人者，至少用调试模式验证。
