# SPEC PHASE 06: UI、体验、音效与游戏反馈打磨

## 阶段目标

在核心玩法和联机可运行的基础上，提升可读性、操作感和派对氛围。此阶段不改变核心规则，只做表现和体验优化。

## 功能范围

### 1. Lobby UI

- 游戏标题；
- 创建房间；
- 加入房间；
- 玩家昵称；
- 简短玩法说明入口。

### 2. Room UI

- 房间码明显显示；
- 玩家列表；
- 准备状态；
- 分享按钮；
- 开始按钮；
- 返回 Lobby；
- 网络状态。

### 3. Game HUD

找人者 HUD：

- 阶段；
- 倒计时；
- 剩余拍击次数；
- 已抓人数；
- 当前分数。

躲藏者 HUD：

- 阶段；
- 倒计时；
- 当前道具名称或图标；
- 是否已被抓；
- 当前分数。

### 4. Preview 表现

- 显示“观察地图，记住道具位置”。
- 角色不出现。
- 地图清晰可见。
- 倒计时明显。

### 5. Hide 阶段找人者屏保

找人者看到：

- 黑屏或静态美术屏保；
- “躲藏者正在布置现场”；
- 倒计时；
- 不显示地图。

### 6. 拍击反馈

- 扇形挥击动画；
- 普通道具破碎效果；
- 抓到躲藏者的强调反馈；
- 拍空反馈；
- 拍击次数用完提示。

### 7. 伪装反馈

- 切换道具时外观立即改变。
- 不加夸张烟雾。
- 轻微的 UI 图标变化即可。
- 搜索阶段移动速度明显偏慢。

### 8. Result 面板

本轮结算显示：

- 找人者抓到几人；
- 哪些躲藏者存活；
- 本轮得分变化；
- 总分排名；
- 下一轮找人者。

整场结算显示：

- 总排名；
- MVP 标签，可选：最会藏、最会抓。

### 9. 音效

至少需要：

- 按钮点击；
- 倒计时；
- 切换道具；
- 拍击；
- 道具破碎；
- 抓获；
- 回合开始；
- 回合结束；
- 胜利/失败。

---

## 可读性要求

- 找人者和躲藏者 UI 颜色或布局要明显不同。
- 剩余拍击次数必须一眼可见。
- 倒计时最后 5 秒要有强调。
- 被抓玩家要能明确知道自己已被抓。
- 观战或被抓后不要误以为还能操作。

---

## 验收标准

- [ ] 新玩家 1 分钟内能理解基本操作。
- [ ] 找人者不会误以为自己有其他技能。
- [ ] 躲藏者不会误以为可以进入容器。
- [ ] 拍击范围和反馈清晰。
- [ ] 得分变化清楚。
- [ ] Hide 阶段找人者完全看不到地图。
- [ ] UI 在不同屏幕尺寸下不遮挡核心玩法。
